# 文档目录说明

本目录保存 PullerBox iOS 的项目约定、架构规范、产品阶段计划、UI 设计指导和工程实现指导。

## 顶层文档

- `00-overview.md`：项目技术栈和开发约定概览。
- `01-architecture.md`：项目架构规范。所有工程实现都必须遵循该文档。

## 子目录

- `ui/`：面向 UI 设计师，说明特性的用户目标、页面结构、主要流程和状态设计方向。
- `engineering/`：面向开发工程师和 AI 开发代理，说明特性的领域模型、业务规则、数据流、算法和测试要求。
- `plans/`：保存跨 UI 与工程的阶段范围、路线图、迁移策略和产品决策。

## 阅读建议

开始实现前，先阅读：

1. `00-overview.md`
2. `01-architecture.md`
3. 对应特性的 `plans/` 文档
4. 对应特性的 `engineering/` 文档

开始设计前，先阅读：

1. 对应特性的 `plans/` 文档
2. 对应特性的 `ui/` 文档
