# 训练指标 UI 设计指南

本文档面向 UI 设计师，描述训练指标在摘要、详情和趋势页面中的用户意义、展示优先级、可视化方式和空值状态。

相关工程算法见 `docs/04-training-metrics-engineering-guide.md`。

## 设计目标

第一版训练指标围绕指力训练，尤其是单手负重悬吊。指标要回答训练者真正关心的问题：

- 我的最大力量是多少。
- 最大力量水平总共能维持多久。
- 我什么时候开始明显力竭。
- 力竭后还能稳定维持多少力量。
- 从最大力量下降了多少。
- 从最大力量下降到力竭后低点用了多久，下降速度多快。
- 经过计划内休息后，我平均能恢复到最大力量的多少。

## 展示原则

### 按动作展示

力量、力竭和恢复指标必须按动作分别展示，不做跨动作混合的主指标。

示例：

```text
左手悬吊
  最大力量 40.1kg
  最大力量维持 28s
  力竭开始 01:10
  平均恢复 82%

右手悬吊
  最大力量 42.5kg
  最大力量维持 34s
  力竭开始 未检测到
  平均恢复 91%
```

如果同一个动作在同一训练计划中出现多次，UI 合并为同一个动作指标组，不按计划位置拆分。

训练摘要顶部可以显示完成信息，例如：

```text
已完成 2 个动作 · 18 reps · 训练时间 12:40
```

但不要把不同动作的力量指标混成一个“本次最大力量”主卡片。

### 记录名称和趋势名称

- 记录详情显示训练当时快照中的动作名称。
- 趋势和汇总显示当前动作名称。
- 如果历史名称发生过变化，显示轻量提示：

```text
该动作历史名称曾变化
```

### timerOnly 状态

未连接拉力计完成的训练仍显示动作完成情况和时间，但力量指标为空。

建议文案：

```text
未记录力量指标
本次训练未连接拉力计，仅保存完成情况和时间。
```

### 空值展示

指标为空时不要显示 0。使用 `--` 或明确状态文案。

常见空值：

- 未连接拉力计。
- 没有有效 work 样本。
- 未检测到力竭。
- 力竭后未检测到稳定维持片段。
- 趋势有效点少于 2 个。

## 核心指标

第一版核心指标共 7 个。UI 默认优先展示前 4 个，后 3 个放在详情或展开区域。

默认展示：

1. 最大力量
2. 最大力量维持时长
3. 力竭开始时间
4. 平均恢复程度

详情展示：

5. 力竭后维持力量
6. 力量下降幅度
7. 力量下降速率

## 指标 1：最大力量

### 用户意义

回答：

```text
这次训练中，这个动作最高达到多少力量？
```

它是其他疲劳和恢复指标的参照基准。

### 展示建议

摘要主卡显示：

```text
最大力量
42.5kg
```

详情页可以在曲线中标出最大力量水平线。

### 说明文案

```text
本次锻炼阶段达到的最高力量水平，已过滤瞬时尖峰。
```

### 空值

- timerOnly：不显示数值。
- 无有效力量样本：`--`。

## 指标 2：最大力量维持时长

### 用户意义

回答：

```text
接近最大力量的状态，全程累计维持了多久？
```

它不是单次最长维持，而是整个训练中接近最大力量的累计时间。

### 展示建议

摘要主卡显示：

```text
最大力量维持
68s
```

详情页可以展示各 rep 中接近最大力量的片段分布。

### 说明文案

```text
全程接近本次最大力量的累计时间。
```

### 空值

- 无最大力量：`--`。
- 无达到最大力量区间的样本：`0s` 可以显示，因为这表示确实没有累计维持时间。

## 指标 3：力竭开始时间

### 用户意义

回答：

```text
从什么时候开始，力量明显低于本次最大力量水平？
```

### 展示建议

摘要主卡显示：

```text
力竭开始
01:24
```

未检测到时：

```text
力竭开始
未检测到
```

详情页建议在力量曲线上标出力竭开始点。

### 说明文案

```text
力量持续明显低于本次最大力量时，标记为力竭开始。
```

## 指标 4：平均恢复程度

### 用户意义

回答：

```text
经过计划内休息后，这个动作平均能恢复到最大力量的多少？
```

### 展示建议

摘要主卡显示：

```text
平均恢复
86%
```

详情页显示恢复趋势：

```text
92% -> 88% -> 81% -> 76%
```

可以用折线图或按休息事件列表展示。

### 说明文案

```text
该动作每次计划内休息后，下一次锻炼恢复到本次最大力量的比例。
```

### 空值

- 训练中没有计划内休息：`--`。
- 休息后没有下一个 work 段：不生成恢复事件。
- 动作第一次出现前的准备或换手时间不算恢复。

## 指标 5：力竭后维持力量

### 用户意义

回答：

```text
明显力竭之后，最低还能稳定维持在什么力量水平？
```

### 展示建议

详情卡显示：

```text
力竭后维持力量
31.8kg
```

曲线中可以标出产生该值的稳定窗口。

### 说明文案

```text
力竭后稳定片段中的最低维持力量。
```

### 空值

- 未检测到力竭：`--`，说明“未检测到力竭”。
- 力竭后没有稳定片段：`--`，说明“未检测到稳定维持片段”。

## 指标 6：力量下降幅度

### 用户意义

回答：

```text
从最大力量到力竭后维持力量，下降了多少？
```

### 展示建议

详情卡主显百分比，辅显 kg：

```text
下降幅度
-22% · -8.4kg
```

如果没有有效下降：

```text
下降幅度
0% · 未出现有效下降
```

### 说明文案

```text
本次最大力量与力竭后维持力量之间的差值。
```

### 空值

- 未检测到力竭：`--`。
- 无力竭后维持力量：`--`。

## 指标 7：力量下降速率

### 用户意义

回答：

```text
从最大力量下降到力竭后低点，用了多久，下降速度多快？
```

下降过程可能跨多个 reps，因此用训练流程时间计算，而不是只统计 work 时间。

### 展示建议

详情卡显示：

```text
下降用时
04:30

下降速率
-0.03kg/s
```

### 说明文案

```text
从首次进入最大力量区间，到力竭后最低稳定片段之间的下降速度。时间不包含暂停。
```

### 空值

- 未检测到力竭：`--`。
- 无力竭后维持力量：`--`。
- 下降用时无效：`--`。

## 摘要页建议

摘要页按动作分组。每个动作默认显示 4 个主指标：

```text
动作名称
最大力量
最大力量维持
力竭开始
平均恢复
```

点击动作或展开后显示：

```text
力竭后维持力量
下降幅度
下降用时
下降速率
恢复事件列表
```

如果力量指标为空，动作卡显示完成情况和空状态说明，不展示空指标网格。

## 详情页建议

详情页围绕单个动作展示：

- 力量曲线。
- 最大力量水平线。
- 最大力量维持片段。
- 力竭开始点。
- 力竭后最低稳定窗口。
- 恢复事件列表。
- 7 个指标的完整数值。

曲线标注应避免过度拥挤。默认只标出最大力量、力竭开始和力竭后稳定窗口；其他信息可通过点击或列表展示。

## 趋势页建议

趋势的基本单位是：

```text
训练计划 - 动作 - 指标
```

趋势点必须来自：

- 同一训练计划内容版本。
- 同一动作。
- 同一指标。
- 指标值非空。

UI 支持两个入口，放在不同页面：

### 按动作看趋势

```text
计划：最大指力训练
动作：左手悬吊

最大力量
最大力量维持
力竭开始
平均恢复
```

### 按指标看趋势

```text
计划：最大指力训练
指标：最大力量

左手悬吊
右手悬吊
```

默认趋势：

- 最大力量。
- 最大力量维持时长。
- 力竭开始时间。
- 平均恢复程度。

可选趋势：

- 力竭后维持力量。
- 力量下降幅度。
- 力量下降速率。

### 趋势计划版本说明

趋势页面主显计划名称，不显示 signature 或 hash。

当同一计划存在多个内容版本时，可以显示：

```text
当前内容版本 · 12 次记录
旧内容版本 · 8 次记录
```

页面说明：

```text
趋势仅比较训练内容一致的记录。修改计划内容后会形成新的趋势。
```

如果有效点少于 2 个：

```text
记录不足，无法展示趋势
```
