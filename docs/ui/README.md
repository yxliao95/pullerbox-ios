# UI 文档说明

本目录面向 UI 设计师，描述每个特性的设计目标、用户流程、页面目的和关键状态。

## 写作边界

- 每个文档对应一个特性。
- UI 文档表达设计理念和页面结构，不规定具体布局、组件样式、视觉细节或完整文案。
- 影响业务状态、数据有效性、训练执行、保存逻辑和引用关系的规则，以 `../engineering/` 中的工程文档为准。
- 阶段范围、暂不包含内容和迁移策略，以 `../plans/` 中的路线图文档为准。
- 新增特性时，应同时检查是否需要对应的工程文档和路线图文档。
