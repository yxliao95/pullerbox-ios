# 计划训练（Planned Training）UI 设计指南

相关文档：

- 路线图：`../plans/planned-training-roadmap.md`
- 工程设计：`../engineering/planned-training-engineering-guide.md`

本文档面向 UI 设计师，描述计划训练（Planned Training）的用户目标、主要页面、核心流程和状态设计方向。具体布局、组件样式、视觉表达和最终文案由 UI 设计师根据产品体验自行把握。

## 设计目标

计划训练的核心体验是让用户围绕一个训练计划完成闭环：

```text
动作库
  -> 训练计划库
  -> 计划编辑
  -> 训练执行
  -> 训练摘要
  -> 保存或丢弃结果
```

设计应帮助用户在训练前知道“能不能开始”、训练中知道“现在该做什么”、训练后知道“本次完成得如何”。

## 核心概念

- 动作（Action）：可复用的训练模板，例如左手悬挂、右手悬挂。
- 训练计划（TrainingPlan）：由计划步骤组成的训练内容。
- 动作组（ActionGroup）：训练计划内部的循环结构，可包含动作和间隔。
- 计划间隔（IntervalStep）：准备、换手、休息等独立倒计时阶段。
- 拉力计模式（forceDevice）：训练开始时已连接拉力计。
- 计时模式（timerOnly）：训练开始时未连接拉力计，只执行倒计时。

## 主要页面

### 训练首页

训练首页以当前训练计划为中心。用户应能快速判断：

- 当前是否有训练计划。
- 是否已选择当前计划。
- 当前计划是否有效。
- 是否已连接拉力计。
- 下一步应该开始、选择、创建还是修复计划。

首页需要覆盖这些状态：

- 没有训练计划。
- 有计划但未选择当前计划。
- 已选择有效计划。
- 已选择无效计划。
- 设备已连接或未连接。

### 动作库

动作库用于管理可复用动作（Action）。设计重点是让用户能轻松新增、编辑、删除动作，并理解动作被计划引用时可能产生影响。

### 动作编辑

动作编辑用于创建或修改一个动作（Action）。第一版动作以 timed reps 为主，包含名称、次数、锻炼时长和组内休息时长。

当动作已被训练计划引用时，界面需要帮助用户理解修改会影响哪些计划，并提供继续修改或另存为新动作的路径。

### 训练计划库

训练计划库用于查看、选择、编辑和删除训练计划（TrainingPlan）。无效计划也应保留可见，并明确提示其不可开始训练。

当前计划应有清晰标识。删除当前计划后，用户需要重新选择或创建计划。

### 计划编辑

计划编辑用于组装训练计划。设计重点是表达清楚层级关系：

```text
训练计划
  -> 计划步骤
    -> 动作组
      -> 组内步骤
```

动作（Action）和计划间隔（IntervalStep）应在视觉上容易区分。循环次数和组间休息属于动作组（ActionGroup），不属于单个动作。

### 训练执行

训练执行页应尽量聚焦当前阶段，减少非必要信息。用户最需要看到：

- 当前阶段。
- 当前阶段剩余时间。
- 当前动作。
- 当前 Set / Rep。
- 动作组进度。
- 计划总进度。

拉力计模式（forceDevice）下应显示当前力量和阶段曲线。计时模式（timerOnly）下只需要清楚呈现倒计时和当前训练内容。

### 暂停状态

暂停状态需要让用户知道训练已停止推进，以及如何恢复或结束。

设备断连导致的意外暂停应明显区别于用户主动暂停，避免用户误以为训练仍在正常进行。

### 训练摘要

训练摘要用于让用户确认本次训练结果，并明确选择保存或丢弃。摘要应先呈现完成情况，再根据测量模式展示力量相关信息。

计时模式（timerOnly）完成的训练仍应展示完成情况和时间，但不展示力量指标。

## 主要流程

### 选择计划并训练

```text
进入训练首页
  -> 选择训练计划
  -> 可选连接拉力计
  -> 开始训练
  -> 执行训练计划
  -> 完成或手动结束
  -> 查看训练摘要
  -> 保存或丢弃
  -> 返回训练首页
```

### 自定义训练计划

```text
进入训练首页
  -> 打开训练计划库
  -> 新建或编辑计划
  -> 添加动作组
  -> 添加动作或计划间隔
  -> 设置循环和休息
  -> 保存计划
```

### 管理动作库

```text
进入动作库
  -> 新建动作
  -> 编辑动作
  -> 删除动作
```

## 状态设计重点

- 计划为空、未选择计划、计划无效和计划有效是训练首页的核心状态。
- 无效计划不应被隐藏，用户需要能看见并修复。
- 未连接拉力计不是开始训练的阻断条件。
- 训练执行页应区分 work、rep rest、interval、group rest、paused 和 resume countdown。
- 摘要页应明确表达记录尚未保存。
- 力量相关内容只在拉力计模式（forceDevice）下展示。
