# 训练指标（Training Metrics）UI 设计指南

相关文档：

- 路线图：`../plans/training-metrics-roadmap.md`
- 工程设计：`../engineering/training-metrics-engineering-guide.md`

本文档面向 UI 设计师，描述训练指标（Training Metrics）的用户意义、页面入口和展示方向。具体图表样式、卡片布局、交互细节和最终文案由 UI 设计师自行更新。

## 设计目标

训练指标应帮助用户理解一次指力训练的表现，尤其是单手负重悬吊中的力量、疲劳和恢复情况。

第一版指标主要回答：

- 最大力量达到多少。
- 接近最大力量的状态维持了多久。
- 什么时候开始明显力竭。
- 力竭后还能稳定维持多少力量。
- 从最大力量下降了多少。
- 下降过程用了多久、下降速度多快。
- 休息后平均恢复到最大力量的多少。

## 展示原则

### 按动作展示

力量、力竭和恢复指标应按动作（Action）分别展示，不把不同动作混成一个主指标。

如果同一个动作在同一次训练计划中出现多次，UI 可以合并为同一个动作指标组。

### 尊重训练模式

计时模式（timerOnly）训练没有力量采样。UI 应展示完成情况和时间，但不要展示空的力量指标网格。

拉力计模式（forceDevice）训练可以展示力量、疲劳、恢复和趋势相关指标。

### 空值不显示为 0

没有数据、未检测到力竭、没有稳定片段、趋势点不足等情况，不应显示为 0。UI 可以使用空状态或轻量说明表达原因。

## 核心指标

第一版核心指标共 7 个：

1. 最大力量（maxForce）
2. 最大力量维持时长（maxForceHoldDuration）
3. 力竭开始时间（fatigueStart）
4. 平均恢复程度（averageRecoveryRatio）
5. 力竭后维持力量（postFatigueSustainedForce）
6. 力量下降幅度（forceDrop）
7. 力量下降速率（forceDropRate）

摘要页面优先展示前 4 个指标。详情或展开区域可以展示后 3 个指标和恢复事件。

## 摘要页

摘要页按动作分组展示。每个动作先呈现完成情况，再呈现核心力量指标。

当某个动作没有力量指标时，动作卡仍应能表达训练完成情况，例如完成 reps、训练时间或计时模式说明。

## 详情页

详情页围绕单个动作展示，适合承载：

- 力量曲线。
- 最大力量水平。
- 最大力量维持片段。
- 力竭开始点。
- 力竭后稳定窗口。
- 恢复事件列表。
- 7 个指标的完整解释。

图表标注应避免过度拥挤。默认只突出最能帮助用户理解表现变化的标记。

## 趋势页

趋势的基本单位是：

```text
训练计划 - 动作 - 指标
```

趋势页面应让用户理解：只有训练内容一致的记录才适合放在同一条趋势中比较。修改计划内容后，会形成新的趋势版本。

趋势入口可以支持两种视角：

- 按动作看多个指标。
- 按指标看多个动作。

有效趋势点少于 2 个时，应展示记录不足状态。
