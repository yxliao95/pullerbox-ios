# 用户流程

本文档汇总当前文档中已经确定的核心用户流程，面向产品、设计和开发协作使用。

依据：

- `01-architecture.md`

## 成功标准

用户流程应支持用户完成一个完整闭环：先创建或选择训练内容，再开始并执行训练，训练结束后查看未保存摘要，最后保存或丢弃训练结果并回到训练首页。

## 核心对象

- 动作（Action）：可复用的训练模板，例如左手悬挂、右手悬挂。
- 训练计划（TrainingPlan）：由计划步骤组成的训练内容。
- 动作组（ActionGroup）：训练计划内部的循环结构，可包含动作和自定义倒计时。
- 自定义倒计时（CustomCountdown）：用户在训练计划或动作组中手动添加的倒计时步骤，例如准备、换手、休息。
- rep 间休息（rep rest）：动作内部两次 rep 之间按动作参数自动产生的休息阶段。
- 组间休息（group rest）：动作组两轮 cycle 之间按动作组参数自动产生的休息阶段。
- 拉力计模式（forceDevice）：训练开始时已连接拉力计。
- 计时模式（timerOnly）：训练开始时未连接拉力计，只执行倒计时。

## 训练内容关系

训练计划的完整结构、字段范围、有效性和签名规则见 `03-training-plan.md`。本文只保留用户流程中需要理解的关系。

训练计划由计划步骤组成。计划步骤可以是自定义倒计时，也可以是动作组。

动作组只存在于训练计划内部，不是独立库对象。动作组内部包含组内步骤、循环次数和组间休息。组内步骤可以继续包含动作或自定义倒计时。

自定义倒计时、rep 间休息和组间休息都表现为倒计时，但来源不同：自定义倒计时由用户显式加入计划；rep 间休息来自动作参数；组间休息来自动作组参数。界面文案和信息层级应帮助用户区分它们。

动作是独立库对象。训练计划引用动作 ID，不直接复制动作内容。开始训练时，系统解析动作引用并生成本次训练的只读执行快照，确保训练开始后本次内容不再被后续编辑影响。

## 总流程

用户通常从训练首页开始。首页根据当前状态引导用户创建计划、选择计划或修复无效计划；动作库作为训练内容的基础入口，用于维护可复用动作。

当用户拥有一个有效的当前计划后，可以开始训练。开始训练时，系统根据拉力计连接状态确定测量模式：已连接时进入拉力计模式，未连接时进入计时模式。

训练执行过程中，用户按计划完成各阶段，也可以主动暂停、恢复或手动结束。训练可能自然完成，也可能因用户手动结束，或在意外暂停后由用户选择结束。

训练结束后，系统进入未保存训练摘要。用户可以保存记录并返回训练首页，也可以确认丢弃结果后返回训练首页。取消丢弃时，用户留在摘要页。

## 训练首页流程

训练首页以当前训练计划为中心。用户进入后应能快速判断：有没有计划、是否已选择当前计划、当前计划是否有效、设备是否已连接，以及下一步能做什么。

- 计划库为空时，首页主操作是新建计划。
- 计划库非空但未选择当前计划时，首页主操作是选择计划，次操作是新建计划。
- 已选择计划但计划无效时，首页显示计划名称和无效原因，主操作是修复计划，次操作是选择其他计划；开始训练不可用。
- 已选择有效计划时，首页显示计划名称、动作数量、预计训练时间和设备状态，主操作是开始训练，次入口包括编辑计划、切换计划、计划库和动作库。

未连接拉力计不是开始训练的阻断条件。未连接时点击开始训练，直接进入计时模式。

## 创建和维护训练内容

### 动作库

动作库支持新建、编辑和删除动作。

动作被训练计划引用时，编辑或删除前需要让用户理解影响范围。编辑已被引用动作时，应提供保存并更新计划、另存为新动作、取消等路径。删除已被引用动作后，相关计划保留可见但变为无效。

### 训练计划库

训练计划库支持新建、编辑、删除计划，并选择当前计划。

训练计划库需要显示所有计划，包括无效计划。无效计划不可开始训练，但应可编辑修复。

新建计划保存后，新计划成为当前计划。编辑当前计划后，当前计划保持不变。编辑非当前计划后，不自动切换当前计划。删除当前计划后，系统清空当前选择，不自动切换到其他计划。

### 计划编辑

计划编辑由三层内容组成：训练计划包含计划步骤；计划步骤可以是自定义倒计时，也可以是动作组；动作组内部继续包含组内自定义倒计时和动作。

设计重点是让层级关系清楚：动作和自定义倒计时应容易区分；循环次数和组间休息属于动作组，不属于单个动作。

计划保存时需要满足最小有效条件：至少包含一个动作组，至少包含一个有效动作引用，每个动作组至少包含一个有效动作，循环次数至少为 1。编辑过程中可以出现临时无效状态，但保存和开始训练前必须回到有效状态。

自定义倒计时可以出现在训练计划或动作组步骤的开头、结尾或中间。连续自定义倒计时不自动合并，应分别作为多个自定义倒计时阶段展示和执行。组间休息只发生在动作组 cycle 之间，不发生在最后一轮之后；rep 间休息只发生在 rep 之间，不发生在最后一次 rep 之后。

## 开始训练

用户点击开始训练后，系统先校验当前计划是否有效，再解析动作引用并生成只读训练执行快照。随后系统根据设备连接状态确定测量模式，并进入计划的第一个步骤。

训练开始前没有系统级准备倒计时。如果用户需要准备时间，应在训练计划开头添加一个自定义倒计时。

训练开始后：

- 不允许修改计划或动作参数。
- 不允许调整剩余 reps、cycles、work、rest。
- 不允许跳过当前阶段。
- 测量模式固定，训练中不切换。

计时模式不能在训练中切换为拉力计模式。拉力计模式断连后进入意外暂停，不降级为计时模式。

## 训练执行流程

训练执行页聚焦当前阶段，让用户知道“现在该做什么”。

训练执行页需要覆盖 work、rep rest、自定义倒计时（customCountdown）、group rest、paused 和 resume countdown 等状态。其中 customCountdown 对应用户添加的自定义倒计时，rep rest 对应动作内部休息，group rest 对应动作组轮次之间的休息。

训练中只支持暂停、恢复和结束三个操作。

手动结束训练需要确认。确认后进入未保存摘要页。

拉力计模式下，work、rep rest、自定义倒计时和 group rest 阶段可以显示阶段曲线；paused 阶段不绘制曲线，应显示静止暂停画面。work 阶段是主要动作表现阶段，休息和自定义倒计时阶段可以弱化显示，避免用户误以为这些阶段也参与主要动作统计。

## 暂停、恢复和断连

暂停分为用户主动暂停和设备断连造成的意外暂停。用户主动暂停后进入 paused 状态；拉力计模式下设备断连时进入意外暂停，恢复前必须重新连接设备。

恢复时先进入 3 秒 resume countdown，再回到训练阶段。

如果 work 阶段被中断：

系统丢弃当前 work 阶段数据，当前 rep 不算完成，恢复后重新开始该 work 阶段。

如果非 work 阶段被暂停：

系统保留该阶段剩余时间，恢复后继续剩余倒计时。

恢复倒计时计入总时间，但不计入训练时间，也不参与动作统计。拉力计模式下暂停期间可以继续保存采样，但采样应标记为 paused，并默认不参与训练统计。

## 训练结束和摘要

结束原因：

- completed：计划自然完成。
- stoppedByUser：用户手动结束。
- stoppedAfterUnexpectedPause：意外暂停后用户选择结束。

训练结束后，系统生成摘要并展示未保存摘要页。用户选择保存时，系统保存训练记录并返回训练首页。用户选择不保存时，需要先确认丢弃；确认后丢弃结果并返回训练首页。

摘要页应明确表达记录尚未保存。取消丢弃时，用户留在摘要页。

## 摘要展示规则

训练摘要需要区分计划预计时长、总时间、训练时间和暂停时长：

- 计划预计时长：训练开始时根据执行快照静态计算，不包含暂停、意外暂停、恢复倒计时和提前结束造成的差异。
- 总时间：从训练开始到结束的墙钟时间，包含暂停、意外暂停和恢复倒计时。
- 训练时间：实际执行训练内容的时间，不包含暂停、意外暂停和恢复倒计时。
- 暂停时长：总时间减去训练时间。

一次 action step 在某个 cycle 中完整执行一次，算该动作完成 1 组。同一个动作在计划中出现多次时，摘要主视图按唯一 action ID 合并统计。

提前结束时，摘要应能区分完整完成的组、已经开始但未完整完成的组、完整完成的 reps，以及被中断的 rep。

所有测量模式都展示：

- 总时间。
- 训练时间。
- 暂停时长。
- 包含动作数。
- 按动作统计的完成组数。
- 按动作统计的未完成组数。
- 按动作统计的完成 reps。
- 按动作统计的组间休息时间。

拉力计模式额外展示：

- 当前训练保存的力量采样。
- 最大力量。
- rep 级力量统计。
- 可用于训练指标计算的动作表现数据。

计时模式不展示：

- 当前力量。
- 最大力量。
- 曲线。
- rep peak。
- 力量类训练指标。

## 设计检查点

- 首页状态必须区分：无计划、未选择、计划无效、计划有效。
- 无效计划不隐藏，用户需要能看到并修复。
- 未连接拉力计可以开始训练，只是进入计时模式。
- 训练执行页必须清楚区分 work、rep rest、自定义倒计时、group rest、paused、resume countdown，尤其要区分用户添加的自定义倒计时和系统按动作参数产生的休息阶段。
- 主动暂停和设备断连造成的意外暂停要明显不同。
- 未保存摘要页必须让用户明确选择保存或丢弃。
- 力量曲线和力量指标只在拉力计模式下出现。
