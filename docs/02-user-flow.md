# 用户流程

本文档是产品设计入口文件，用来描述一个个独立闭环的预期用户流程。具体功能规则放在后续专题文档中维护。当前关联文档：

- 训练计划、动作、动作组、自定义倒计时、有效性、执行快照：见 `03-training-plan.md`。
- 训练摘要、力量指标、趋势和空值口径：见 `04-training-metric.md`。

## 设计原则

每个用户流程都应形成闭环。闭环定义为：用户带着一个明确目标进入，经过必要操作、反馈和确认后，能到达一个明确结束状态，并知道下一步可以做什么。

本文档中的流程只保留产品体验层面的关键判断。若某个要点需要详细字段、计算规则、边界条件或工程约束，应拆到独立文档，并在对应流程中引用。

## 流程总览

当前产品的核心流程可以理解为一个训练流程的两个分支：

- 制定计划：用户创建或维护可执行的训练内容，最终得到一个可执行的有效计划。
- 执行训练：用户基于当前计划完成一次训练，最终保存或丢弃本次训练结果。

两条分支互相衔接：

```text
训练首页
  ├─ 没有可执行计划 -> 制定计划 -> 回到训练首页
  └─ 已有有效当前计划 -> 执行训练 -> 保存或丢弃结果 -> 回到训练首页
```

训练首页是两条流程的连接点。它需要让用户快速判断当前处于哪种状态：

- 没有任何计划。
- 有计划，但未选择当前计划。
- 已选择当前计划，但计划无效。
- 已选择有效当前计划，可以开始训练。

## 流程 1：制定计划

### 流程目标

用户把可复用动作组织成一套可执行、可保存、可复现的训练计划。

成功结束状态：

- 用户创建或编辑了一个训练计划，并将计划保存后返回原页面。

### 适用入口

用户可能从这些位置进入制定计划流程：

- 训练首页没有计划时，点击新建计划。
- 当前计划无效时，点击修复计划。
- 用户主动进入计划库、计划编辑或动作库。

### 主路径

1. 用户从训练首页进入计划库，或直接创建新训练计划。
2. 用户创建或选择训练计划。
3. 用户维护计划内容，包括动作组、动作和自定义倒计时。
4. 系统在保存或开始训练前校验计划有效性。
5. 用户保存有效计划。
6. 系统返回训练首页，并展示当前计划的名称、预计训练时间、动作概览和设备状态。

训练计划的结构、字段范围、有效性和预计时长规则见 `03-training-plan.md`。

### 分支：维护动作库

动作是计划编排的基础对象。用户可以在制定计划过程中进入动作库，创建、编辑或删除动作。

需要关注：

- 动作可能被多个训练计划引用。
- 编辑已被引用动作时，应让用户理解影响范围。
- 删除已被引用动作后，相关计划不应被隐藏，而应保留可见并进入无效状态。
- 无效计划需要提供修复路径。

动作引用、影响范围和无效计划规则见 `03-training-plan.md`。

### 分支：修复无效计划

当当前计划或计划库中的某个计划无效时，用户应能看到无效原因，并进入编辑流程修复。

需要关注：

- 无效计划不可开始训练。
- 无效计划仍应显示在计划库中。
- 缺失动作位置应支持替换动作或移除此步骤。
- 当前计划变无效时，首页保留当前选择，并显示修复入口。

### 设计要点

- 制定计划流程的核心不是执行训练，而是产出一个有效计划。
- 计划级步骤、动作组内部步骤、动作和自定义倒计时之间的层级必须清楚。
- 自定义倒计时是用户显式添加的计划内容，不等同于 rep 间休息或组间休息。
- 编辑过程中可以出现临时无效状态，但保存和开始训练前必须回到有效状态。
- 如果计划规则变复杂，应优先更新 `03-training-plan.md`，本文只保留入口、路径和关键体验要求。

## 流程 2：执行训练

### 流程目标

用户基于当前有效计划完成一次训练，训练结束后查看未保存摘要，并选择保存或丢弃本次训练结果。

成功结束状态：

- 用户保存训练记录并回到训练首页。
- 或用户确认丢弃训练结果并回到训练首页。

### 适用入口

用户从训练首页点击开始训练进入该流程。

开始训练前必须满足：

- 已选择当前计划。
- 当前计划有效。

未连接拉力计不是阻断条件。未连接时开始训练，进入计时模式。

### 主路径

1. 用户在训练首页确认当前计划。
2. 用户点击开始训练。
3. 系统校验计划有效性，并生成本次训练的只读执行快照。
4. 系统根据设备连接状态进入拉力计模式或计时模式。
5. 用户按计划执行训练阶段。
6. 计划自然完成后，系统进入未保存训练摘要。
7. 用户查看摘要。
8. 用户保存记录，系统返回训练首页。

执行快照、测量模式和训练计划展开规则见 `03-training-plan.md`。

训练摘要和指标展示规则见 `04-training-metric.md`。

### 分支：计时模式训练

当用户开始训练时未连接拉力计，系统进入计时模式。

需要关注：

- 计时模式仍可完成训练流程。
- 计时模式仍可保存训练记录。
- 计时模式展示时间、完成组数、完成 reps 等结构化结果。
- 计时模式不展示当前力量、最大力量、力量曲线和力量类指标。
- 计时模式训练中不切换为拉力计模式。

### 分支：训练中暂停和恢复

训练中只支持暂停、恢复和结束三个操作。

暂停分为：

- 用户主动暂停。
- 拉力计模式下设备断连造成的意外暂停。

需要关注：

- 主动暂停和意外暂停应有明显不同的状态反馈。
- 意外暂停恢复前必须重新连接设备。
- 恢复时先进入 5 秒恢复倒计时，再回到训练阶段。
- work 阶段被中断时，当前 work 阶段数据丢弃，当前 rep 不算完成，恢复后重新开始该 work。
- 非 work 阶段被暂停时，保留剩余时间，恢复后继续倒计时。

暂停、恢复倒计时和力量指标是否参与统计的口径见 `04-training-metric.md`。

### 分支：手动结束训练

用户可以在训练中主动结束训练。手动结束需要确认。

确认后：

- 系统结束本次训练。
- 系统生成未保存摘要。
- 摘要中需要表达这是提前结束的训练。
- 用户仍然可以选择保存或丢弃。

提前结束时，摘要应能区分完整完成的组、已经开始但未完整完成的组、完整完成的 reps，以及被中断的 rep。详细指标和摘要口径见 `04-training-metric.md`。

### 分支：丢弃训练结果

训练结束后，记录尚未保存。用户可以选择不保存本次训练结果。

需要关注：

- 摘要页必须明确表达记录尚未保存。
- 丢弃需要二次确认。
- 确认丢弃后返回训练首页。
- 取消丢弃时留在摘要页。

### 训练阶段呈现

训练执行页聚焦当前阶段，让用户知道现在该做什么。

需要覆盖的阶段包括：

- work。
- rep rest。
- 自定义倒计时 customCountdown。
- group rest。
- paused。
- resume countdown。

设计重点：

- work 是主要动作表现阶段。
- rep rest 来自动作参数。
- group rest 来自动作组参数。
- customCountdown 来自用户显式添加。
- 休息和自定义倒计时阶段可以弱化力量表现，避免用户误以为它们参与主要动作统计。

### 设计要点

- 执行训练流程的核心不是编辑内容，而是完成一次训练并处理结果。
- 开始训练后，本次训练内容不可修改。
- 测量模式在训练开始时确定，训练中不切换。
- 拉力计模式断连后进入意外暂停，不降级为计时模式。
- 训练结束后先进入未保存摘要，由用户明确选择保存或丢弃。
- 如果摘要、指标或趋势规则变复杂，应优先更新 `04-training-metric.md`，本文只保留流程中的展示和决策点。

## 训练首页状态

训练首页是制定计划和执行训练的共同入口，应根据当前状态提供不同主操作。

| 状态 | 用户看到 | 主操作 | 次操作 |
| --- | --- | --- | --- |
| 没有计划 | 空状态和创建引导 | 新建计划 | 动作库 |
| 有计划但未选择 | 计划库摘要或选择提示 | 选择计划 | 新建计划 |
| 当前计划无效 | 计划名称和无效原因 | 修复计划 | 选择其他计划 |
| 当前计划有效 | 计划名称、预计时长、动作概览、设备状态 | 开始训练 | 编辑计划、切换计划、计划库、动作库 |

未连接拉力计时，开始训练仍可用，但首页应提示将进入计时模式。

## 文档拆分规则

当流程中的某个要点开始包含大量细节时，应拆成独立文档。

适合留在本文的内容：

- 用户目标。
- 流程入口和结束状态。
- 主路径。
- 分支路径。
- 状态反馈。
- 阻断条件。

适合拆出去的内容：

- 数据结构。
- 字段范围。
- 计算公式。
- 有效性规则。
- 指标定义。
- 趋势签名。
- 存储结构。
- 工程实现细节。

当前拆分方向：

- `03-training-plan.md` 负责训练内容本身。
- `04-training-metric.md` 负责训练结果和指标。

## 设计检查点

- 02 文档是否能作为产品设计入口，让开发者先理解用户要完成哪些闭环流程。
- 每个流程是否都有清楚的入口、目标、主路径、分支和结束状态。
- 训练和制定计划是否被区分为两个流程，而不是混在一个功能清单中。
- 复杂规则是否已经指向后续专题文档，而不是在本文重复展开。
- 训练首页是否清楚承接两条流程。
- 无效计划是否可见、可修复且不可开始训练。
- 未连接拉力计是否可以开始训练，并明确进入计时模式。
- 训练结束后是否必须经过未保存摘要，并由用户选择保存或丢弃。
