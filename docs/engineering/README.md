# Engineering 文档说明

本目录面向开发工程师和 AI 开发代理，描述每个特性的领域模型、业务规则、数据流、算法、持久化口径和测试要求。

## 写作边界

- 每个文档对应一个特性。
- 工程文档必须遵循 `../01-architecture.md`。
- 工程文档应尽可能具体，足以指导实现和测试。
- UI 表现、视觉风格和页面细节由 `../ui/` 文档表达。
- 阶段范围、暂不包含内容和迁移策略，以 `../plans/` 中的路线图文档为准。
- 当业务规则影响状态机、数据有效性、保存逻辑、引用关系或训练执行行为时，应写入工程文档。
